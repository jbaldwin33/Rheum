ii.	Based on requirements from document titled CCS4455/CS6457 Final Project	Grading	Criteria

3D/GameFeel:
Game is 3D Game Feel game
Player can solve all of the puzzles and beat the final boss to beat the game
The player can be killed and respawn
Start menu is present
The player can reset the game through the start menu by going back to the main menu
Game is in third person

Precursors to Fun Gameplay:
Player is provided with different ways to fight enemies, and must solve puzzles
Game features switches, moving platforms, etc
Harder puzzles later in the game

Skeletal 3D mesh character:
Character uses blendtree with multiple layers
Character turning is programmatic, and animation adjusts to match character movement
No snapping in animations
Camera is smooth, and will attempt to avoid clipping
Footsteps, landing sounds, and many others are present

3D world with physics...:
Created unique environment
All significant actions have auditory and graphical feedback
Appropriate boundaries implemented
Environmental physical interactions are present, pushable blocks, reflecting projectiles
3D simulation with six degrees movement is present

Real time NPC:
Multiple NPCs including spiders, zombie soldiers, and the final boss
NPCs animation matches movement, same method as used for player
Particles and audio feedback implemented for AI

Polish:
Footstep sounds and particles implemented, differing for sand/stone
Cohesive aesthetic, unified color pallet

Immersiveness:
Game feels fun/immersive

v.
Controls:
WASD move
Space jump
Shift roll
Left click attack
Right click block
Hold left click	jump attack

Play through game, test combat, solve puzzles.

Tips:
Your shield can reflect certain projectiles
Certain attacks can do damage through the shield, such as the jump attack
As you delve deeper into the dungeon, watch your sleep meter and see how the environment changes
